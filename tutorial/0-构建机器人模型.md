# 0-构建机器人模型

官方参考文档：https://nvlabs.github.io/curobo/latest/getting-started/build_robot_model.html

本节目标是从自定义机械臂 URDF 出发，生成 cuRobo 可用的 robot model 配置文件：

- `mec_arm_from_urdf.yml`：cuRobo 原生 robot config
- `mec_arm_from_urdf.xrdf`：Isaac Sim / Isaac Lab 生态更常用的 robot description 格式

cuRobo 不能只依赖普通 URDF 做运动规划。URDF 主要描述 link、joint、mesh、惯量和关节限制，但 cuRobo 还需要两类额外信息：

- 每个 link 的 collision spheres，用于快速碰撞检测
- self-collision ignore matrix，用于跳过相邻 link 或不需要检查的自碰撞对

cuRobo 官方教程中的 `RobotBuilder` 会从 URDF 和 mesh 自动拟合 collision spheres，并生成 self-collision ignore matrix。

## Step 1: Prepare your URDF

你的机械臂 URDF 路径是：

```text
/home/light/workspace/arm_vla/mec_arm_model/urdf/mec_arm.urdf
```

URDF 中的 mesh 路径使用 `package://` 格式，例如：

```xml
<mesh filename="package://mec_arm_model/meshes/base_link.STL" />
<mesh filename="package://mec_arm_model/meshes/Empty_Link1.STL" />
```

cuRobo 的 builder 会自动去掉 `package://` 前缀。去掉之后，它需要从下面这个相对路径找到 mesh：

```text
mec_arm_model/meshes/base_link.STL
```

所以 `--asset-path` 应该指向这个相对路径的父目录，也就是项目根目录：

```text
/home/light/workspace/arm_vla
```

本机械臂中真实带 mesh 的 link 是：

```text
base_link
Empty_Link1
Empty_Link2
Empty_Link3
Empty_Link4
Empty_Link5
Empty_Link6
```

这些 link 会被用于拟合 collision spheres。

URDF 中还有一些辅助 frame/link：

```text
Joint4_zero_link
Joint5_zero_link
Joint6_zero_link
TCP_link
```

其中 `TCP_link` 是末端工具坐标系，用来表示规划目标位姿。它不是实体碰撞体，也不需要 collision spheres。

因此构建 cuRobo robot model 时，需要显式指定：

```bash
--tool-frames TCP_link
```

## Step 2: Run the tutorial

在项目根目录执行：

```bash
cd /home/light/workspace/arm_vla

PYTHONPATH=/home/light/workspace/curobo:${PYTHONPATH:-} \
/data/conda_envs/isaacsim51_3dgs_grasp/bin/python \
  -m curobo.examples.getting_started.build_robot_model \
  --urdf /home/light/workspace/arm_vla/mec_arm_model/urdf/mec_arm.urdf \
  --asset-path /home/light/workspace/arm_vla \
  --output /home/light/workspace/arm_vla/mec_arm_sim/configs/curobo/mec_arm_from_urdf.yml \
  --export-xrdf \
  --tool-frames TCP_link \
  --sphere-density 1.0 \
  --num-collision-samples 1000 \
  --compute-metrics \
  --seed 42
```

参数说明：

```text
--urdf
  输入机械臂 URDF。

--asset-path
  mesh 资源根目录。因为 URDF 里写的是 package://mec_arm_model/meshes/...，
  所以这里应该是 /home/light/workspace/arm_vla。

--output
  输出 cuRobo 原生 YAML 配置。

--export-xrdf
  同时导出 XRDF。

--tool-frames TCP_link
  指定末端工具坐标系。

--sphere-density 1.0
  collision spheres 密度。第一版先用默认密度，后续再调。

--num-collision-samples 1000
  用 1000 个随机关节采样生成 self-collision ignore matrix。

--compute-metrics
  打印每个 link 的 sphere fitting 质量。

--seed 42
  固定随机种子，保证结果可复现。
```

如果要可视化拟合出来的 collision spheres，在同一条命令末尾追加：

```bash
--visualize
```

## Step 4: Understand the pipeline

官方 builder 主要做三件事。

第一，sphere fitting。

cuRobo 会把每个 link 的 mesh 近似成若干 collision spheres。后续碰撞检测主要基于这些球体，而不是直接对复杂三角 mesh 做实时距离查询。这样可以大幅提高规划速度。

第二，self-collision matrix。

机械臂中很多 link 对不需要每次都检查碰撞。例如相邻 link 通常天然接触或距离很近，如果不忽略，会导致规划器认为机械臂一直处于自碰撞状态。builder 会通过随机采样关节状态，生成一份 self-collision ignore matrix。

第三，export。

builder 会把 URDF 解析出的运动学树、joint limits、tool frame、collision spheres 和 self-collision ignore matrix 写入配置文件。YAML 是 cuRobo 原生格式，XRDF 更适合和 Isaac Sim / Isaac Lab 工作流对接。

## Step 5: Tuning

第一版先保留默认参数：

```bash
--sphere-density 1.0
--num-collision-samples 1000
```

后续如果 collision spheres 可视化效果不好，再调这些参数：

```text
--sphere-density
  提高或降低每个 link 的碰撞球数量。数值越大，覆盖更细，但规划计算也更重。

--coverage-weight
  提高 mesh 内部覆盖程度。覆盖不足时可以调大。

--protrusion-weight
  降低球体伸出 mesh 外部的程度。球体外扩太严重时可以调大。

--clip-link base_link z 0.0
  如果 base_link 的球体穿过安装平面或桌面，可以裁剪 base_link 的 z 方向球体。

--num-collision-samples
  增加 self-collision matrix 采样数量。复杂机械臂或自碰撞结果不稳定时可以调大。
```

当前阶段不要急着调参。先完成：

```text
1. 确认 joint_names 顺序等于 Isaac Sim DOF order
2. 用 mec_arm_from_urdf.yml 做 cuRobo config load check
3. 再进入 IK / MotionPlanner feasibility 调试
```
